## 🔧 解决步骤

### 方法 1: 使用自动诊断脚本（推荐）

```bash
cd /Users/mac/Desktop/project/myapplication
./check_microphone.sh
```

脚本会自动检查所有配置，并启动实时监控。

### 方法 2: 手动配置检查

#### 步骤 1: 启用模拟器麦克风

**重要：这是最关键的一步！**

1. 在模拟器窗口右侧找到 **"..."** 按钮（三个点）
2. 点击打开扩展控制面板
3. 选择 **"Settings"** （不是其他选项）
4. 找到 **"Microphone"** 部分
5. ✅ **勾选**: "Virtual microphone uses host audio input"

```
Extended controls
├── Settings
│   ├── General
│   ├── Proxy
│   └── Microphone  👈 在这里
│       └── ☑️ Virtual microphone uses host audio input  👈 确保勾选
```

#### 步骤 2: 授予 macOS 系统麦克风权限

**macOS Sequoia (15.x):**
1. 打开 **系统设置**
2. **隐私与安全性** → **麦克风**
3. 找到并启用：
   - ✅ **Android Emulator**
   - ✅ **qemu-system-aarch64**
   - ✅ **Android Studio** (如果使用)

**如果列表中没有这些应用**：
- 首次使用时会弹出权限请求，点击"允许"
- 如果错过了，需要重启模拟器触发权限请求

#### 步骤 3: 在应用中测试

1. 启动应用
2. 在屏幕底部找到调试面板（往上移动了一些）
3. 点击 **"测试麦克风"** 按钮
4. **对着电脑麦克风清晰地说话 5 秒**
5. 查看测试结果

**成功的输出**：
```
✅ 麦克风工作正常! 最大音量: 2543
```

**失败的输出**：
```
❌ 麦克风无信号，音量为0
```

如果失败，继续下一步。

#### 步骤 4: 检查日志

```bash
# 启动日志监控
adb logcat -s MicrophoneTest MainActivity VoiceRecognitionService

# 点击"测试麦克风"，应该看到：
# MicrophoneTest: AudioRecord初始化成功
# MicrophoneTest: 开始录音测试...
# MicrophoneTest: 读取音频数据: 3584 字节, 音量: XXX
```

**如果看到 "音量: 0"**：
- 模拟器麦克风未启用
- macOS 未授予权限
- 电脑麦克风被占用

#### 步骤 5: 重启模拟器

有时需要完全重启模拟器才能使麦克风配置生效：

```bash
# 关闭模拟器
adb emu kill

# 等待 5 秒
sleep 5

# 重新启动（跳过快照）
emulator -avd Medium_Phone_API_36.1 -no-snapshot-load &

# 等待启动完成
adb wait-for-device

# 重新安装应用
./gradlew installDebug

# 启动应用
adb shell am start -n com.example.myapplication/.MainActivity
```

### 方法 3: 使用真实 Android 设备（最可靠）

如果模拟器问题难以解决，使用真实手机是最快的方案：


## 🔍 常见错误和解决方案

### 错误 1: "音量为 0"

**原因**: 模拟器未接收到麦克风输入

**解决**:
1. 检查模拟器麦克风设置（步骤1）
2. 检查 macOS 权限（步骤2）
3. 重启模拟器（步骤5）
4. 关闭其他占用麦克风的应用

### 错误 2: "AudioRecord 初始化失败"

**原因**: 麦克风权限问题或硬件不可用

**解决**:
```bash
# 手动授予权限
adb shell pm grant com.example.myapplication android.permission.RECORD_AUDIO

# 重启应用
adb shell am force-stop com.example.myapplication
adb shell am start -n com.example.myapplication/.MainActivity
```

### 错误 3: "客户端错误"（语音识别）

**原因**: Google 语音识别服务问题

**说明**:
- 这个错误不影响麦克风测试工具
- 麦克风测试工具使用的是 AudioRecord（不需要 Google 服务）
- 如果麦克风测试成功，说明硬件没问题
- 语音识别错误是 Google 服务的问题，在真实设备上通常不会出现

## 💡 重要提示

1. **模拟器麦克风配置是最常见的问题**
   - 确保 "Virtual microphone uses host audio input" 已勾选
   - 配置后需要重启模拟器

2. **macOS 权限必须授予**
   - 首次会弹出权限请求
   - 如果误点拒绝，需要在系统设置中手动添加

3. **麦克风测试工具 vs 语音识别**
   - 麦克风测试工具直接读取音频，不依赖 Google 服务
   - 即使语音识别失败，麦克风测试也可能成功
   - 先确保麦克风测试成功，再处理语音识别问题

4. **真实设备最可靠**
   - 如果时间紧迫，直接用真机测试
   - 真机上不需要任何特殊配置
   - 语音识别在真机上通常 100% 可用
