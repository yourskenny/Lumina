## 🔧 使用方法

### 1. 语音识别

应用会自动监听语音命令（需要麦克风权限）：

1. 打开应用
2. 授予麦克风权限
3. 直接说出命令
4. 系统自动执行并语音反馈结果

### 2. 按钮点击

如果语音识别不可用，所有功能都可以通过按钮操作：

1. 点击右下角的浮动菜单按钮 (☰)
2. 在侧边抽屉中选择功能
3. **快捷功能区** - 14个用户功能按钮
4. **开发者工具区** - 6个测试调试按钮

---

## 🌍 语言设置

### 自动切换

应用会自动检测语言包：

- **中文可用**: 优先使用中文识别 (zh-CN)
- **中文不可用**: 自动切换到英文 (en-US)

### 手动切换

在侧边抽屉的"开发者工具"区域：

1. 点击"切换语言"按钮
2. 在中文和英文之间切换
3. TTS会播报当前语言

## 🐛 故障排查

### 真机识别器忙碌错误

如果在真机上遇到 `ERROR_RECOGNIZER_BUSY`：

1. **重启设备** - 清理系统资源
2. **清除应用数据**:
   ```bash
   adb shell pm clear com.example.myapplication
   ```
3. **检查其他应用** - 关闭占用麦克风的应用
4. **等待熔断恢复** - 应用会在10秒后自动重试

### 模拟器语音识别

模拟器可能遇到的问题：

- ❌ **中文语言包缺失** - 使用英文命令
- ⚠️ **识别延迟** - 正常现象
- ✅ **按钮功能正常** - 可以直接点击按钮

### 语音无法识别

如果说了命令但没有反应：

1. **检查音量** - 确保声音足够大
2. **检查语言** - 使用对应语言的命令
3. **查看状态卡片** - 抽屉顶部显示监听状态
4. **使用按钮** - 作为备选方案

## 📊 技术细节

### 语音识别引擎

- **引擎**: Android SpeechRecognizer API
- **在线识别**: 需要网络连接
- **语言支持**: zh-CN (中文), en-US (英文)
- **自动重启**: 识别完成后500ms自动重启

### 错误处理

- **连续错误计数**: 最多5次
- **熔断机制**: 5次后暂停10秒
- **智能延迟**: 根据错误类型调整
  - ERROR_RECOGNIZER_BUSY: 3秒 + 销毁重建
  - ERROR_CLIENT: 2秒
  - 其他错误: 1秒

### TTS反馈

所有命令执行都有语音反馈：

- **命令名称**: 先播报命令
- **执行结果**: 播报操作结果
- **错误提示**: 紧急播报错误信息

## 💡 提示和技巧

### 最佳实践

1. **清晰发音** - 说话清晰，吐字清楚
2. **安静环境** - 减少背景噪音
3. **标准用语** - 使用表格中的标准命令
4. **确认反馈** - 听到TTS反馈确认执行

### 推荐命令

最常用的简短命令：

- 中文: "拍照"、"暂停"、"继续"、"查询电池"
- 英文: "photo"、"pause"、"resume"、"battery"

### 组合使用

可以连续使用多个命令：

```
说: "拍照"  → 等待反馈 → 说: "切换摄像头" → 等待反馈 → 说: "拍照"
```
