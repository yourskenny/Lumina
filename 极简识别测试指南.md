# 🔍 极简识别测试指南

## 问题诊断

你遇到的情况：
- ✅ Android 系统语音助手可以识别语音
- ✅ 我们的应用能检测到声音（onBeginningOfSpeech）
- ❌ 但最终总是返回"未识别到语音"错误

这说明：
- 硬件正常（麦克风工作）
- Google 服务正常（系统助手能用）
- **问题在于我们应用的识别配置**

## 🆕 新增功能：极简测试

我添加了一个"极简测试"按钮，它使用**最基本的识别配置**，不添加任何额外参数。

### 关键改进

1. **默认使用英文识别**
   - 英文语言包更可靠
   - 离线中文包在模拟器中不可用（error 12）

2. **移除了可能导致问题的参数**
   - 移除了自定义超时参数
   - 移除了语言偏好设置
   - 使用系统默认配置

3. **极简测试模式**
   - 只设置最基本的 LANGUAGE_MODEL
   - 不设置任何额外参数
   - 完全依赖系统默认行为

## 📊 错误码解释

根据系统日志，可能的错误：

| 错误码 | 名称 | 含义 | 解决方案 |
|--------|------|------|---------|
| 7 | ERROR_NO_MATCH | 听到声音但无法识别 | 说得更清楚，或切换语言 |
| 6 | ERROR_SPEECH_TIMEOUT | 未检测到声音 | 提高音量，离麦克风近些 |
| 5 | ERROR_CLIENT | 客户端错误 | 等待后重试 |
| 8 | ERROR_SERVER | 服务器错误 | 检查网络连接 |

## 🔍 详细诊断

### 如果极简测试成功

说明：
- ✅ 基本识别功能正常
- ✅ 问题在于之前的额外配置参数

**下一步**：
- 继续使用当前的简化配置
- 可以尝试添加中文识别（但要谨慎）

### 如果极简测试也失败

错误可能是：

**1. ERROR_NO_MATCH (最常见)**
```
# 查看详细日志
adb logcat -s NetworkSpeechRecognizer SodaSpeechRecognizer

# 可能看到：
# Failed to get language pack of required locale: error 12
# Speech recognition error type NO_SPEECH_DETECTED
```

**原因**：
- 说话不够清晰
- 音量太小
- 语言不匹配（设置英文却说中文）

**解决**：
- 说清楚的英文单词："hello", "test", "yes"
- 离麦克风近一点
- 音量正常（不要太小声）

**2. ERROR_NETWORK (网络错误)**

**检查网络**：
```bash
adb shell ping -c 3 8.8.8.8
```

**3. ERROR_SPEECH_TIMEOUT (超时)**

说明麦克风没接收到声音，返回步骤1检查麦克风配置。

## 💡 对比分析

### 系统助手 vs 我们的应用

**系统语音助手使用**：
- Intent: com.google.android.googlequicksearchbox
- 直接调用 Google 服务
- 有完整的 UI 和反馈

**我们的应用使用**：
- SpeechRecognizer API
- RecognizerIntent
- 后台服务，无 UI

**关键差异**：
- 系统助手可能有特殊权限
- 系统助手使用优化的配置
- 我们需要精确配置参数

## 📄 技术细节

### 极简配置代码
```kotlin
val simpleIntent = Intent(RecognizerIntent.ACTION_RECOGNIZE_SPEECH).apply {
    putExtra(RecognizerIntent.EXTRA_LANGUAGE_MODEL,
             RecognizerIntent.LANGUAGE_MODEL_FREE_FORM)
    // 不设置任何其他参数，使用系统默认
}
```

### 标准配置代码
```kotlin
val intent = Intent(RecognizerIntent.ACTION_RECOGNIZE_SPEECH).apply {
    putExtra(RecognizerIntent.EXTRA_LANGUAGE_MODEL,
             RecognizerIntent.LANGUAGE_MODEL_FREE_FORM)
    putExtra(RecognizerIntent.EXTRA_LANGUAGE, "en-US")
    putExtra(RecognizerIntent.EXTRA_CALLING_PACKAGE, context.packageName)
    putExtra(RecognizerIntent.EXTRA_PARTIAL_RESULTS, true)
    putExtra(RecognizerIntent.EXTRA_MAX_RESULTS, 3)
    putExtra(RecognizerIntent.EXTRA_PREFER_OFFLINE, false)
}
```

### 区别
- 极简：只设置必需的参数
- 标准：添加了语言、包名、在线模式等配置
