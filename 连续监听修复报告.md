# 连续监听修复报告

## 📋 问题描述

### 用户报告的Bug

1. **模拟器上无法持续监听** - 应用只显示"准备就绪"后就停止监听
2. **可以识别语音但不连续** - 能识别一次但之后不再监听
3. **需要优化** - 希望识别过程不中断，实现一直监听

---

## 🔍 问题诊断

### 初始日志分析

```
02-02 00:25:40.100  准备好接收语音
02-02 00:25:40.289  开始说话
02-02 00:25:41.440  尝试重新启动识别器...
02-02 00:25:41.440  已在监听中  ← 重启被阻止！
```

**问题点**：重启时提示"已在监听中"导致无法继续。

### 代码深入分析

#### 问题1: `isListening` 状态管理错误

**错误代码**（修复前）：
```kotlin
override fun onResults(results: Bundle?) {
    // ...
    onDebugInfo?.invoke(VoiceDebugInfo(
        text = "✓ $text",
        isListening = true  // ❌ 错误！识别已完成
    ))

    // 自动重启监听
    if (shouldRestart) {
        CoroutineScope(Dispatchers.Main).launch {
            delay(500)
            startListening()  // ❌ 被阻止，因为isListening=true
        }
    }
}

fun startListening() {
    if (isListening) {  // ← 检查失败
        Log.d(TAG, "已在监听中")
        return  // ❌ 直接返回，不重启
    }
    // ...
}
```

**问题流程**：
1. `onReadyForSpeech` → 设置 `isListening = true`
2. 用户说话，识别完成
3. `onEndOfSpeech` → 设置 `isListening = false`
4. `onResults` → **又设置回** `isListening = true` ❌
5. 尝试重启 → `startListening()` 检查到 `isListening = true` → 返回
6. **结果**：停止监听，无法继续

---

#### 问题2: 状态转换时机错误

**错误的状态设置**：
```kotlin
override fun onEndOfSpeech() {
    Log.d(TAG, "结束说话")
    isListening = false  // ❌ 过早设置
}
```

**问题**：
- `onEndOfSpeech` 在 `onResults` 之前调用
- 但有时 `onResults` 会晚一些到达
- 在这期间状态不一致

---

#### 问题3: 重启延迟过长

```kotlin
delay(500)  // ❌ 500ms太长，导致明显的监听空白期
```

---

#### 问题4: 错误类型未区分

```kotlin
consecutiveErrors++  // ❌ 所有错误都计数

// ERROR_NO_MATCH (无法识别) 不应该算严重错误
// 但也被计入，容易触发熔断
```

---

## ✅ 修复方案

### 修复1: 正确管理 `isListening` 状态

**修复后代码**：
```kotlin
override fun onResults(results: Bundle?) {
    val matches = results?.getStringArrayList(SpeechRecognizer.RESULTS_RECOGNITION)
    matches?.firstOrNull()?.let { text ->
        Log.d(TAG, "识别结果: $text")

        onDebugInfo?.invoke(VoiceDebugInfo(
            text = "✓ $text",
            isPartial = false,
            volume = currentVolume,
            isListening = false  // ✅ 正确：识别完成，设置为false
        ))

        processCommand(text)
    }

    // ✅ 关键修复：立即设置为false以允许重启
    isListening = false

    // 自动重启监听
    if (shouldRestart) {
        CoroutineScope(Dispatchers.Main).launch {
            delay(200)  // ✅ 减少延迟
            if (shouldRestart) {
                Log.d(TAG, "🔄 自动重启监听...")
                startListening()
            }
        }
    }
}
```

**改进**：
- ✅ 在 `onResults` 中明确设置 `isListening = false`
- ✅ 确保重启前状态正确
- ✅ 减少延迟到200ms

---

### 修复2: 优化状态转换

**修复后代码**：
```kotlin
override fun onEndOfSpeech() {
    Log.d(TAG, "结束说话 - 等待识别结果...")
    // ✅ 不在这里设置isListening = false
    // 让onResults或onError来管理状态
}
```

**理由**：
- `onEndOfSpeech` 只是说话结束，但识别还在进行
- 状态应该由最终结果（`onResults` 或 `onError`）来设置
- 避免状态不一致

---

### 修复3: 区分严重和非严重错误

**修复后代码**：
```kotlin
// 根据错误类型决定是否增加错误计数
val isSeriousError = error !in listOf(
    SpeechRecognizer.ERROR_NO_MATCH,      // 无法识别
    SpeechRecognizer.ERROR_SPEECH_TIMEOUT  // 语音超时
)

if (isSeriousError) {
    consecutiveErrors++
    Log.e(TAG, "严重错误，计数: $consecutiveErrors")
} else {
    Log.d(TAG, "非严重错误（无匹配/超时），不计入错误次数")
}

// 根据错误类型决定延迟时间
val delayTime = when (error) {
    SpeechRecognizer.ERROR_RECOGNIZER_BUSY -> {
        Log.w(TAG, "识别器忙碌，彻底重新初始化...")
        speechRecognizer?.destroy()
        speechRecognizer = null
        3000L  // 识别器忙碌：3秒
    }
    SpeechRecognizer.ERROR_CLIENT -> {
        2000L  // 客户端错误：2秒
    }
    SpeechRecognizer.ERROR_NO_MATCH,
    SpeechRecognizer.ERROR_SPEECH_TIMEOUT -> {
        200L  // ✅ 无匹配/超时：快速重启
    }
    else -> 1000L  // 其他错误：1秒
}
```

**改进**：
- ✅ `ERROR_NO_MATCH` 和 `ERROR_SPEECH_TIMEOUT` 不计入错误次数
- ✅ 这些错误200ms快速重启，保持连续监听
- ✅ 避免误触发熔断机制

---

### 修复4: 增强错误恢复

**修复后代码**：
```kotlin
fun startListening() {
    if (isListening) {
        Log.w(TAG, "⚠️ 已在监听中，跳过重启 (isListening=$isListening)")
        return
    }

    // ✅ 如果识别器为null，重新初始化
    if (speechRecognizer == null) {
        Log.d(TAG, "识别器为null，重新初始化...")
        initializeSpeechRecognizer()
    }

    shouldRestart = true
    val intent = createRecognitionIntent()
    speechRecognizer?.startListening(intent)
    Log.d(TAG, "✓ 开始监听语音命令 - 语言: $currentLanguage")
}
```

**改进**：
- ✅ 自动检测并重建损坏的识别器
- ✅ 详细日志便于调试

## 📝 技术细节

### 状态机设计

```
[停止]
  ↓ startListening()
[等待就绪] (isListening=false)
  ↓ onReadyForSpeech()
[监听中] (isListening=true)
  ↓ onBeginningOfSpeech()
[说话中] (isListening=true)
  ↓ onEndOfSpeech()
[处理中] (isListening=true)
  ↓ onResults()
[识别完成] (isListening=false) ← 关键修复
  ↓ delay(200ms)
[停止] → 循环回到开始
```

### 错误分类策略

| 错误类型 | 严重性 | 计数 | 延迟 | 动作 |
|---------|--------|------|------|------|
| ERROR_NO_MATCH | 低 | 否 | 200ms | 快速重启 |
| ERROR_SPEECH_TIMEOUT | 低 | 否 | 200ms | 快速重启 |
| ERROR_CLIENT | 中 | 是 | 2000ms | 延迟重启 |
| ERROR_RECOGNIZER_BUSY | 高 | 是 | 3000ms | 重建识别器 |
| ERROR_NETWORK | 高 | 是 | 1000ms | 延迟重启 |

### 重启时间对比

| 场景 | 修复前 | 修复后 | 改进 |
|------|--------|--------|------|
| 识别成功 | 不重启 ❌ | 200ms ✅ | ∞ |
| 无法识别 | 不重启 ❌ | 200ms ✅ | ∞ |
| 语音超时 | 不重启 ❌ | 200ms ✅ | ∞ |
| 客户端错误 | 2000ms | 2000ms | - |
| 识别器忙碌 | 3000ms | 3000ms + 重建 | 更可靠 |
